	Привет! Приятно, что хоть кто-то кроме пользователей
будет пользоваться этим инструментом! Надеюсь, ты останешься доволен

	Установка исходников для Ubuntu написана в файле instruction.txt,
после устновки можно следовать данному readme
	
	Для начала необходимо разархивировать этот проект в какую-нибудь 
папку и запустить settings.exe, чтобы инициализировать базу данных, установить 
путь к ней и записать прокси. Все эти данные фиксируются в файле settings.json.
Можно ввести свои параметры, но также можно использовать значения задаваемые 
программой по умолчанию.

	После этого для использования приложения необходимо получить токен 
телеграм бота. Для этого необходимо написать в телеграме боту @BotFather команду 
/newbot, ввести имя бота и запомнить токен.
	
	Как только токен получен необходимо открыть командную строку и с помощью 
команды cd перейти в папку client проекта и запустить main.exe. Программа получает на 
вход несколько параметров, которые сейчас разберем. Для примера используем рабочий 
токен одного уже созданного бота @BMSTUnewsletter_bot: 1143632014:AAFhp3pxu5jjnWfot1VfMKAXeWRnCHhQ9mU

-- Запуск бота: main.exe start 1143632014:AAFhp3pxu5jjnWfot1VfMKAXeWRnCHhQ9mU file1.json
	- start - команда запуска
	- 1143632014:AAFhp3pxu5jjnWfot1VfMKAXeWRnCHhQ9mU - токен бота
	- file1.json - путь к файлу, задаваемый фразы бота, поля файла можно менять по 
		своему усмотрению, ничего не задав, программа использует фразы по умолчанию
		!файл необходимо перевести в кодировку utf-8
		бот  поддерживает следующие настройки фраз: 	greetings   - приветствие
								approval    - поддтверждение регистрации
								unsubscribe - реакция на отписку
								advice      - совет по началу работы
								change_name - реакция на смену имени
	данная команда запускает бота в системе, который может регистрировать новых пользователей

-- Рассылка сообщений: main.exe BMSTUnewsletter Коля Леша Катя "Завтра зачет!"
	- BMSTUnewsletter - имя бота
	- Коля Леша Катя - имена зарегистрированных пользователей
	- "Завтра зачет!" - сообщение (для нескольких слов необходимо передавать сообщение в кавычках)
	данная команда совершает рассылку конкретного сообщения введенным пользователям

-- Остановка бота: main.exe stop BMSTUnewsletter
	- stop - команда запуска
	- BMSTUnewsletter - имя бота
	данная команда завершает работу бота

-- Копирование базы данных: main.exe backup [путь до базы данных в которую необходимо копировать]
	- backup - команда копирования
	- путь до базы данных в которую необходимо копировать - необязательный параметр
	данная команда копирует содержимое текущей базы данных в базу данных по умолчанию, 
	либо в конкретную, выбранную вами

-- Откат базы данных: main.exe rollback [путь, откуда взять содержимое]
	- rollback - команда отката
	- путь, откуда взять содержимое - необязательный параметр
	данная команда откатывает состояние текущей базы данных к раннее сохраненному, либо 
	к той, которую вы ввели


	Рассмотрены все основные команды обеспечивающие желанную работоспособность приложения. 
В случае непредвиденных ошибок, прежде всего необходимо остановить бота прописав соответствующую 
команду. Затем можно продолжить работу приложения.

	Нельзя запускать одного бота, пока работает бот с таким же токеном.
	
	После работы, необходимо завершить работу бота, иначе могут возникнуть ошибки.

	Важное отступление! В связи с текущей проблемой, а именно: блокировка телеграмма на территории
РФ было принято решение использовать прокси при работе с ботом. На данный момент оно работает, однако
в случае если прокси перестанет работать, есть следующие варианты:
	1) Заменить прокси) Как вы помните, оно задается в файле settings.json папки db
	2) Использовать vpn. При разработке бота отлично себя показал windscribe. Это не реклама, 
хотя мы были бы рады оплате)
